# üèóÔ∏è 8.1 Classes & Objects (OOP Basics)

Python is an **object-oriented programming (OOP) language**.
OOP helps you **organize code into reusable blueprints** called **classes**.

---

## üîë Key Concepts

* **Class** ‚Üí blueprint for objects
* **Object** ‚Üí an instance of a class
* **Attributes** ‚Üí variables inside a class
* **Methods** ‚Üí functions inside a class

---

## üìå 1. Defining a Class

```python
class Person:
    # Attribute
    species = "Human"

    # Constructor / initializer
    def __init__(self, name, age):
        self.name = name
        self.age = age

    # Method
    def greet(self):
        print(f"Hello, my name is {self.name} and I am {self.age} years old.")
```

---

## üìå 2. Creating Objects

```python
p1 = Person("Alice", 25)
p2 = Person("Bob", 30)

print(p1.name)  # Alice
print(p2.age)   # 30

p1.greet()      # Hello, my name is Alice and I am 25 years old.
p2.greet()      # Hello, my name is Bob and I am 30 years old.
```

---

## üìå 3. Class vs Object Variables

```python
print(Person.species)  # Human
print(p1.species)      # Human
p1.species = "Alien"
print(p1.species)      # Alien
print(p2.species)      # Human
```

‚úÖ Class variable is shared ‚Üí object variable overrides it.

## üéØ Mini Project: Simple Bank Account

```python
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount
        print(f"Deposited {amount}. New balance: {self.balance}")

    def withdraw(self, amount):
        if amount > self.balance:
            print("‚ùå Insufficient funds")
        else:
            self.balance -= amount
            print(f"Withdrawn {amount}. Remaining balance: {self.balance}")

# Test
acc = BankAccount("Alice", 1000)
acc.deposit(500)
acc.withdraw(300)
acc.withdraw(1500)
```

‚úÖ Output:

```
Deposited 500. New balance: 1500
Withdrawn 300. Remaining balance: 1200
‚ùå Insufficient funds
```

## üß† Best Practices

* Keep attributes **private** using `_` or `__` prefix if needed
* Use **methods** to access or modify attributes
* Use classes to **model real-world objects** in your programs
* Keep code **organized and reusable**
